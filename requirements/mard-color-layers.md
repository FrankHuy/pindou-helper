## 1. 页面分层逻辑（非常值得复用）

这个页面没有把所有颜色混成一个“总数”，而是拆成 3 个查看范围：

1. 完整色号库 · 291  
   - 标准 221 色 + 扩展 70 色
2. 标准系列 · 221  
   - A–H 与 M，共 9 个系列
3. 常见商家套装  
   - 24 / 48 / 72 / 96 / 120 / 144 / 168 / 192 / 216 / 221 / 264  
   - 页面提示：仅筛选本页，不保存选择

页面还单独解释了数字口径：

- 221 / 291：描述色号及系列范围
- 24 到 264：通常描述商家按色卡组合销售的套装
- 商家套装不一定是完整色号库的连续子集

这对我们产品很重要：  
用户买到的不是“官方 291 全套”，而是“商家 72/96/144/168 套装”。  
所以色卡选择器不能只给“全色卡”，必须支持“我手上这套是多少色”。

## 2. 系列结构

完整 291 色，15 个系列：

标准系列（221）

扩展系列（70）

校验：  
26+32+29+26+24+25+21+23+15 = 221

## 4. 数据字段说明

`mard-color-chart.json` 中每个颜色：

```json
{
  "series": "A",
  "code": "A1",
  "hex": "#FAF4C8",
  "rgb": [250, 244, 200],
  "name": "A1"
}
```

注意：

- 源页面只公开了色号 + HEX，没有中文色名
- `name` 目前先用 code 占位
- HEX 是屏幕参考值，不是实体豆绝对真值

## 5. 建议落到产品里的色卡分层

建议 UI / 数据结构直接参考该页面：

### 5.1 第一层：色号范围

- 完整色号库（291）
- 标准系列（221）
- 常见商家套装（24~264）

### 5.2 第二层：套装规格（仅商家套装时显示）

- 24 / 48 / 72 / 96 / 120 / 144 / 168 / 192 / 216 / 221 / 264

### 5.3 第三层：系列筛选

- 全部
- A B C D E F G H M
- 扩展：P Q R T Y ZG

### 5.4 第四层：用户自定义

- 禁用某些颜色（我没有这个色）
- 从当前套装中再减色
- 保存“我的色卡”

## 6. 接入当前项目的建议

当前 `src/lib/palette.ts` 只有约 30 个 MVP 基础色。  
建议下一步：

1. 用 `mard-color-chart.json` 替换/扩展完整色卡数据
2. 用 `mard-merchant-packs.json` 做套装预设
3. palette 数据结构改成：

```ts
type BeadColor = {
  code: string
  name: string
  hex: string
  rgb: [number, number, number]
  series: string
  brand: 'MARD'
}

type PalettePreset = {
  id: string
  label: string
  kind: 'full' | 'standard' | 'merchant'
  size?: number
  codes: string[]
}
```

4. 默认推荐：
   - 新手：商家套装 72 / 96
   - 进阶：144 / 168
   - 专业：标准 221 或完整 291

## 7. 页面交互上值得抄的点

1. 顶部先讲清 291 / 221 / 扩展 70，避免用户误解总数
2. 三个大按钮切换范围，而不是塞进深层菜单
3. 商家套装明确写“仅筛选本页，不保存选择”
4. 系列筛选和套装筛选是正交的
5. 每张色卡同时暴露：
   - 色号可复制
   - HEX 可复制
6. 始终提醒：HEX 是屏幕近似值

## 8. 提取校验

- 完整色号 article 数：291
- 标准系列：221
- 扩展系列：70
- 商家套装 size 与可见色号数一一对应：
  - 24→24 ... 264→264

## 9. 已知限制

1. 没有中文色名
2. HEX 为第三方页面屏幕参考值，后续最好做实物校准
3. 商家套装定义来自 pixel-beads.com 页面筛选逻辑，不是 MARD 官方公开标准
4. 若后续要上生产，建议在 UI 标注数据来源与近似声明
