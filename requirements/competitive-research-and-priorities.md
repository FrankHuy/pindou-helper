# 拼豆图纸生成工具竞品调研与优化建议

日期：2026-07-17
项目：pindou-helper
当前定位：面向 iPhone / iPad / 桌面浏览器的纯前端 PWA 拼豆图纸生成工具，优先支持 MARD 色卡，本地处理图片。

## 1. 当前项目现状

基于当前仓库 README、src/App.tsx、src/lib/pattern.ts、src/lib/palette.ts，项目已有能力如下：

- 上传本地图片，不上传服务器。
- 使用浏览器 Canvas 在本地处理图片。
- 设置图纸宽度，高度按原图比例自动计算。
- 将图片缩放为一格一颗豆的图纸。
- 使用 MARD MVP 基础色卡做最近色匹配。
- 显示/隐藏网格。
- 显示/隐藏色号。
- 显示颜色用量统计。
- 导出 PNG 图纸。
- 已有 PWA 基础配置，适合 iOS/iPadOS 添加到主屏幕。

当前实现特点：

- 色彩匹配算法：加权 RGB 距离。
- 图纸宽度范围：8-160 颗。
- 色卡：src/lib/palette.ts 中约 30 个 MARD MVP 基础色。
- Pattern cell 当前类型为 BeadColor，不支持空格/透明区域。
- 导出格式当前仅 PNG。

当前主要缺口：

- MARD 色卡不完整。
- 没有 PDF / 打印排版。
- 没有分板/分页图纸。
- 没有最大颜色数控制。
- 没有亮度、对比度、饱和度、抖动等转换参数。
- 没有透明背景/背景移除。
- 没有单格编辑器。
- 没有项目保存和制作进度模式。

## 2. 竞品调研

### 2.1 MakeBead

网址：https://makebead.com

定位：免费浏览器版 image-to-bead pattern 工具。

公开信息显示的核心功能：

- 浏览器本地处理图片，不上传服务器。
- 支持 JPEG、PNG、WebP。
- 支持 Perler、Hama、Artkal、Nabbi 多品牌色卡。
- 支持透明 PNG，透明区域会变成空格/无豆。
- 支持板子大小、最大颜色数、亮度、对比度、饱和度、抖动、背景移除等参数。
- 支持 PNG / PDF 导出。
- PDF 包含网格、色号、材料清单。
- 有图纸编辑器：画笔、橡皮、填充、取色、替换全部、撤销/重做。
- 有制作进度模式：标记已完成的豆，本地保存进度。

启发：

MakeBead 的优势不只是能转图，而是能生成真正可制作的图纸。PDF、材料清单、编辑修正、制作进度是核心体验。

### 2.2 Pixel-Beads

网址：https://www.pixel-beads.net

定位：经典免费在线拼豆转换器。

公开信息显示的核心功能：

- 强调 Secure & Private，所有图片处理在浏览器内完成。
- 用户选择图片、豆子数量、颜色数量。
- 支持 Hama、Artkal、Nabbi、Perler。
- 导出 PDF，PDF 中包含开始制作所需信息。

启发：

Pixel-Beads 与当前项目 MVP 路线接近，但它已经把颜色数量限制和 PDF 打印作为基础能力。当前项目只有宽度控制和 PNG 导出，需要补齐可打印工作流。

### 2.3 Perlypop

网址：https://perlypop.com
App Store：https://apps.apple.com/us/app/perlypop-perler-beads/id1450794449

定位：iPhone / iPad / Mac 原生 App，偏专业用户。

公开信息显示的核心功能：

- 支持照片转图纸，也支持手动画图。
- 支持 Perler、Hama、Artkal、Nabbi。
- 支持 Midi / Mini 豆尺寸。
- 照片转换有 wizard，引导用户选择最佳设置。
- 可在图纸上叠加原图作为参考，方便后期整理。
- 支持 PDF 导出。
- 有制作模式：按板子查看、标记单个豆已完成、追踪进度。
- 支持高亮指定颜色，辅助摆豆。

启发：

移动端体验很重要。用户在 iPad 上不是只生成图，还会边看边摆豆。所以制作模式、颜色高亮、按板查看、防迷路能力比单纯算法更能形成差异化。

### 2.4 Beadify / BeadPattern / BeadMapper 等新站

网址：

- https://beadifyapp.com
- https://beadpattern.net
- https://www.beadmapper.com

共同趋势：

- 支持多品牌真实色卡。
- 支持 PDF、购物清单、颜色编号、网格坐标。
- 支持较大图纸，例如 300x300。
- 开始加入 AI 文生图 / 风格生成 / 多预设。
- 提供 Cartoon / Pixel art / Photo / Logo 等转换预设。
- Beadify 展示 Delta E 色差值，用更专业的颜色匹配表达可信度。

启发：

AI 生图可以作为后期功能，但不是第一优先级。拼豆工具的基础信任来自：色卡准不准、打印能不能直接用、用户能不能修图、制作时会不会迷路。

### 2.5 七卡瓦拼豆底稿生成器

网址：https://perlerbeads.zippland.com
GitHub：https://github.com/Zippland/perler-beads

公开信息显示的核心功能：

- 强调素材只在设备上处理，不上传云端。
- 开源。
- 内置多品牌色号体系：MARD、COCO、漫漫、盼盼、咪小窝。
- 支持 168 / 144 / 96 等多种色板规格。
- 使用 CIEDE2000 / Delta E 颜色距离算法，而不是简单 RGB 欧氏距离。

启发：

这是国内方向最强相关竞品，直接覆盖国内拼豆品牌和 MARD。当前项目如果只做上传图片转拼豆，很难形成差异。建议更聚焦 MARD、移动端制作体验、导出排版、项目管理。

## 4. 优化方向

### 方向 A：把“生成图”升级成“可制作的图纸”

建议功能：

1. PDF 导出
   - 首页总览图。
   - 分页大图。
   - 每页带坐标。
   - 色号图例。
   - 颜色用量统计。
   - 图纸尺寸：宽 x 高，总豆数。
   - 可选是否显示色号。

2. 分板/分区打印
   - 按 29x29 或用户自定义板尺寸切块。
   - 每块显示 board index，例如 A1、A2、B1、B2。
   - 每块边缘标注行列坐标。
   - 适合 iPad 查看，也适合 A4 打印。

3. 颜色清单增强
   - 展示 color code、名称、数量、色块。
   - 支持按数量排序/按色号排序。
   - 支持“我已有/缺少”标记。
   - 支持导出清单图片或 CSV。

优先原因：

用户最后要的是能照着摆，不是只看一张像素图。PDF、分板、清单会直接提高实用性。

### 方向 B：提高图片转换质量

建议功能：

1. 最大颜色数量限制
   - 例如 8 / 16 / 24 / 32 / 48 色。
   - 用 k-means / median cut / popularity quantization 先降色，再匹配色卡。
   - 对拼豆非常重要，因为颜色太多会增加购买和制作成本。

2. 图片调整参数
   - 亮度。
   - 对比度。
   - 饱和度。
   - 锐化/边缘增强可后置。
   - 一键预设：照片、头像、卡通、Logo、像素图。

3. Dithering 抖动开关
   - Floyd-Steinberg。
   - Atkinson。
   - 关闭/轻微/强。
   - 对渐变、肤色、照片特别有用。

4. 透明/背景处理
   - 对 PNG alpha 直接支持空格。
   - 支持点击背景色并设置容差，把背景变成空格。
   - 做钥匙扣、摆件时非常实用。

5. 色差算法升级
   - 从当前加权 RGB 升级到 Lab + CIEDE2000。
   - 至少提供“快速 RGB”和“精准色差”两档。
   - 国内开源竞品已经在用 CIEDE2000，建议跟上。

优先原因：

图纸质量是核心体验。尤其拼豆用户常拿头像、动漫图、宠物图，如果效果差，后面的导出再好也没用。

### 方向 C：做真正适合 iPhone/iPad 的“制作模式”

建议功能：

1. 制作模式
   - 全屏显示图纸。
   - 点一下标记已完成。
   - 已完成区域变暗或打勾。
   - 本地保存进度到 localStorage / IndexedDB。
   - 刷新后继续。

2. 按颜色高亮
   - 用户选择一个色号，只高亮该颜色。
   - 其他颜色变淡。
   - 适合“一种颜色一把抓”铺豆方式。

3. 按行/列辅助
   - 当前行高亮。
   - 横纵坐标。
   - 可切换“只看当前板”。

4. 手势体验
   - iPad 双指缩放。
   - 拖动画布。
   - 长按取色。
   - 底部工具栏适配触屏。

优先原因：

当前项目目标明确包含 iPhone/iPad 浏览器/PWA。制作模式是 PWA 相比普通网页转换器的重要差异化。

### 方向 D：补全 MARD 和国内品牌生态

建议功能：

1. 完整 MARD 色卡
   - code、中文名、HEX/RGB、系列、是否常用。
   - 尽量从实物/可信来源校准。
   - 标注“屏幕色仅供参考”。

2. 多套 MARD 色卡规格
   - 例如 96 / 144 / 168。
   - 用户选择自己买的是哪套。
   - 生成时只使用对应色卡。

3. 兼容 COCO / 漫漫 / 盼盼 / 咪小窝
   - 后续再扩。
   - 先把数据结构设计成多品牌可插拔。

4. 自定义色卡/禁用颜色
   - 用户手动禁用某些颜色。
   - 支持“我没有这个色，别用”。
   - 后续可做“替换为最近可用颜色”。

优先原因：

国内用户很看重“我手上的豆能不能直接用”。MARD 是项目天然切入点。

### 方向 E：编辑器能力

建议功能：

- 单格画笔。
- 橡皮擦/空格。
- 取色器。
- 油漆桶填充。
- 替换全部某颜色。
- 撤销/重做。
- 原图半透明叠底参考。
- 局部框选移动/删除，后期再做。

优先原因：

自动转换很少一次完美。用户一定需要修眼睛、边缘、背景、文字。没有编辑器，图纸质量卡在算法上；有编辑器，用户可以自己修。

### 方向 F：项目保存和分享

建议功能：

1. 本地项目保存
   - IndexedDB 保存原图、参数、图纸、编辑记录、制作进度。
   - 支持项目列表。
   - PWA 离线可继续。

2. 导入/导出项目文件
   - JSON 格式。
   - 方便用户备份和迁移。

3. 分享链接/云端图库
   - 不是 MVP 必需。
   - 后期如果要社区增长再做。

优先原因：

项目可能不是一次做完，尤其 100x100 以上的大图。保存和继续制作是高价值功能，但可以排在 PDF/编辑器之后。

## 5. 建议实现优先级

### P0：必须优先做，直接决定产品是否可用

#### P0-1 完整 MARD 色卡 + 色卡数据结构重构

原因：

- 当前只有 MVP 基础色，生成质量和可用性受限。
- 色卡是整个产品的地基。

建议实现：

- 把 palette.ts 改成支持 brand / set / code / name / hex / rgb。
- 支持选择 MARD 基础/完整色卡。
- 后续可无痛增加 COCO、漫漫、盼盼、咪小窝等。

#### P0-2 PDF 导出 + 可打印图纸

原因：

- 竞品几乎都把 PDF 当核心交付物。
- PNG 更适合预览，PDF 才是“可制作”。

建议实现：

- 使用 jsPDF 或 pdf-lib。
- 第一版先做单页总览 + 色表清单。
- 第二版做多页分板。

#### P0-3 最大颜色数控制

原因：

- 拼豆不是普通像素画，颜色越多越难买、越难摆。
- 这是照片转拼豆的关键参数。

建议实现：

- 先做简单 quantization。
- UI 增加“颜色数量：8 / 16 / 24 / 32 / 48 / 不限”。

#### P0-4 图片基础调整

原因：

- 用户上传照片质量差异大。
- 亮度、对比度、饱和度实现成本低，但效果提升明显。

建议实现：

- 在 createPattern 前处理像素。
- 参数变化后防抖重新生成。

### P1：强烈建议下一阶段做，能形成差异化

#### P1-1 CIEDE2000 / Lab 色差匹配

原因：

- 比 RGB 更符合人眼感知。
- 国内开源竞品已经使用这个作为卖点。

建议实现：

- 新增 colorDistanceMode: 'rgb' | 'ciede2000'。
- 默认使用 CIEDE2000。
- 对 palette Lab 值做缓存，减少性能开销。

#### P1-2 透明背景 / 点击背景移除

原因：

- 拼豆作品常需要非矩形边缘。
- 透明区域对应“不放豆”，非常实用。

建议实现：

- PNG alpha 小于阈值时生成 empty cell。
- pattern cell 类型从 BeadColor 改为 BeadColor | null。
- draw/export/counts 都支持空格。

#### P1-3 单格编辑器 MVP

原因：

- 自动图纸需要人工修。
- 没有编辑器，用户会离开去别的软件修。

建议实现：

- 画笔、橡皮、取色。
- 替换全部。
- 撤销/重做使用 history stack。

#### P1-4 制作模式 MVP

原因：

- 和 iPhone/iPad PWA 定位高度匹配。
- 竞品 App 的强功能之一。

建议实现：

- 全屏图纸。
- 点击标记完成。
- 按颜色高亮。
- localStorage 保存 progress bitmap。

### P2：中期增强，提升专业度和留存

#### P2-1 多板/分区视图

原因：

- 大图纸必须分板，不然用户会迷路。

建议实现：

- 默认 29x29。
- 支持自定义板尺寸。
- UI 显示当前板和总板数。

#### P2-2 Dithering 抖动

原因：

- 对照片和渐变有明显帮助，但也可能让图纸变复杂。

建议实现：

- 关闭/轻微/强。
- 先做 Floyd-Steinberg。
- 和最大颜色数联动。

#### P2-3 本地项目保存

原因：

- 大作品不会一次完成。

建议实现：

- 使用 IndexedDB。
- 项目列表。
- 保存原图、参数、pattern、编辑结果、制作进度。

#### P2-4 自定义禁用颜色 / 只用已有颜色

原因：

- 用户常常不是全套豆都有。

建议实现：

- 色卡面板允许禁用颜色。
- 重新生成时排除禁用色。
- 已生成图纸可“替换为最近可用色”。

### P3：后期增长/商业化功能

#### P3-1 AI 文生图 / 风格模板

原因：

- 新竞品在做，但不是基础刚需。
- 成本和复杂度较高，可能需要后端。

建议：

- 等基础图纸工具稳定后再做。
- 可以先做“风格预设”，不急着接 AI。

#### P3-2 社区图库 / 分享链接

原因：

- 有增长价值，但会引入账号、审核、存储、版权问题。

建议：

- 先支持导出项目 JSON。
- 再考虑云端分享。

#### P3-3 购物链接 / 色号购买建议

原因：

- 有商业化价值，但容易受供应链/品牌数据影响。

建议：

- 先做“缺少颜色清单”。
- 后续再接购买链接。

## 6. 推荐近期路线图

### 第 1 阶段：把 MVP 变成真正可用的工具

目标：用户能上传图片，生成一份能打印、能采购、能照着摆的 MARD 图纸。

建议任务：

1. 重构 palette 数据结构，补完整 MARD 色卡。
2. 支持 empty cell，为透明背景和橡皮擦铺路。
3. 增加最大颜色数控制。
4. 增加亮度/对比度/饱和度。
5. 增加 PDF 导出：总览 + 色号清单。
6. 增加分板 PDF：29x29 块。

### 第 2 阶段：提升生成质量和修图能力

目标：让用户能把“不完美的自动结果”修成可用图纸。

建议任务：

1. CIEDE2000 色差。
2. 背景移除。
3. Dithering。
4. 单格编辑器：画笔、橡皮、取色、替换全部。
5. 撤销/重做。

### 第 3 阶段：强化 iPad/PWA 差异化

目标：用户不只是生成图，而是在这个工具里完成制作过程。

建议任务：

1. 制作模式。
2. 按颜色高亮。
3. 标记完成进度。
4. 当前板/当前行辅助。
5. 本地项目保存。

## 7. 如果只选 5 个马上做的功能

建议顺序：

1. PDF 导出
   - 这是从“预览玩具”到“实用工具”的分水岭。

2. 完整 MARD 色卡
   - 拼豆工具的地基，色卡不准其他都白搭。

3. 最大颜色数控制
   - 照片转拼豆如果颜色不可控，会非常难做。

4. 图片调整参数：亮度/对比度/饱和度
   - 实现成本低，效果提升明显。

5. 透明/背景空格支持
   - 拼豆图纸经常不是完整矩形，且后续编辑器依赖 empty cell。

## 8. 产品定位建议

不建议把项目定位成“又一个 image to pixel 工具”。

更建议定位为：

> 面向国内拼豆玩家的 iPhone/iPad 拼豆图纸工作台：本地处理图片，精准匹配 MARD 色号，生成可打印、可分板、可追踪进度的实用图纸。

差异化关键词：

- 国内品牌色卡优先：MARD / COCO / 漫漫 / 盼盼。
- iPad / PWA 制作模式。
- 隐私本地处理。
- 可打印 PDF。
- 分板和进度追踪。
- 适合真实购买和摆豆，而不是只生成好看的像素图。

## 9. 建议转化为需求文档的 Epics

### Epic 1：色卡体系

- 支持多品牌/多规格色卡数据结构。
- 补全 MARD 色卡。
- 支持选择色卡规格。
- 支持禁用颜色。

### Epic 2：图纸生成质量

- 支持最大颜色数。
- 支持亮度/对比度/饱和度。
- 支持透明区域为空格。
- 支持 CIEDE2000。
- 支持 dithering。

### Epic 3：导出与打印

- 支持 PNG 导出增强。
- 支持 PDF 导出。
- 支持分板 PDF。
- 支持颜色清单/材料清单导出。

### Epic 4：图纸编辑

- 支持画笔。
- 支持橡皮擦。
- 支持取色器。
- 支持颜色替换。
- 支持撤销/重做。

### Epic 5：制作模式

- 支持全屏查看。
- 支持按颜色高亮。
- 支持标记完成。
- 支持本地保存进度。
- 支持按板/按行辅助。

### Epic 6：项目保存

- 支持 IndexedDB 本地项目列表。
- 支持恢复上次项目。
- 支持导出/导入项目 JSON。
