# 拼豆工作间：图纸上传与色号高亮

## Goal

新增独立 Tab **「拼豆工作间」**：用户上传一张**已有拼豆图纸图**，系统识别**上方像素格图纸**与**下方用色图例**，把本次用到的豆子做成可点按钮；用户点选某一色号后，图纸中对应像素高亮，便于按色分批拼豆。

与现有 **「拼豆图纸」**（照片 → 生成图纸）互补：工作间消费「已生成/已导出的图纸图」，不做照片量化生成。

## Confirmed product decisions

1. **新 Tab 名称**：拼豆工作间  
2. **核心交互**：上传图纸 → 识别用色 → 色号按钮 → 点击高亮对应像素  
3. **输入范围（B）**：支持**本工具导出 PNG** + **同类第三方导出图**（上格点图纸 + 下图例的常见版式）；**不**把任意手机拍实体图纸 / 杂乱 UI 截图作为 MVP 必达  
4. **上下分割**：优先尝试自动估计图/图例分界；**分界不准时由用户拖动/放置分隔线**（用户已确认可接受）  
5. **色号识别（D，混合，无 OCR）**  
   - 图例区：采样色块 → MARD 最近色 → 生成本图用色按钮（色块 + 色号）  
   - 图案区：格点/像素映射到该用色集合（必要时 fallback 全色卡）  
   - 不做图例文字 OCR；色号为色卡匹配结果，UI 附「屏幕色仅供参考」  
6. **空格处理（A）**：接近白/浅灰底、或与本图用色集合距离都很远的格 → 无豆（`null`）；不进色号按钮；高亮时不当作目标色  
   - 格心/小区域中位色采样，减轻网格线干扰  
7. **处理边界**：浏览器本地 Canvas；不上传图纸到服务器  

## Confirmed from codebase (evidence)

- 顶层 Tab：`bead` | `xhs`（`src/App.tsx`）；切换时 bead 区 `is-hidden` 保状态  
- 现有生成管线已有预览高亮：`highlightCode` + `drawPattern` 非焦点半透明（`pattern.ts`，`HIGHLIGHT_DIM_ALPHA = 0.38`）  
- 本工具 PNG 导出结构：上方 pattern + 下方「用色统计」legend（`exportPattern` / `layoutLegend` / `drawLegend`）  
- 色卡：MARD `BeadColor`（code / hex / rgb），本地处理  
- 无 react-router；新能力宜 `src/features/...` 独立模块  

## Requirements

### Must（随访谈收敛）

- R1. 主界面新增 Tab **拼豆工作间**，可与现有 Tab 切换  
- R2. 支持上传本地图纸图片（png/jpg/webp）  
- R3. 支持本工具导出图与同类「上图下图例」第三方导出；自动估计分界 + **用户可调分隔线**  
- R4. 用色识别采用 **D**：图例色块 → MARD 色号按钮；图案映射到用色集合；**无 OCR**  
- R5. 空格采用 **A**：浅底/远离用色集合 → 无豆；不进按钮列表  
- R6. 点选色号后，图纸区对应像素高亮，其余弱化；再点取消或提供清除  
- R7. 处理全本地，不上传用户图纸  
- R8. `npm run build` / `npm run lint` 通过；现有拼豆生成与 XHS 无回归  
- R9. 色号/色块旁展示 **「屏幕色仅供参考」**  

### Should

- R10. 移动端可用的色号布局与图纸缩放/平移  
- R11. 识别失败时中文错误与可重试（调整分隔线后重识别）  
- R12. 对本工具导出图有较高默认准确率（可作为回归样例）  

### Out of scope (unless later decided)

- 照片 → 生成图纸（已有 Tab）  
- 任意角度手机拍实体板 / 重度透视矫正  
- 图例 OCR 读印刷色号（MVP 不做）  
- 服务端 OCR / 上传存储  
- 多图纸批量、账号云同步  
- 改色后回写导出（非 MVP）  

## Acceptance Criteria（初稿）

- [ ] AC1. 可进入「拼豆工作间」并上传符合「上图下图例」的图纸  
- [ ] AC2. 自动或手动分隔后，用色列表来自图例色块→色卡匹配；空格不进列表  
- [ ] AC3. 点击色号 → 对应像素高亮/突出，其余可读弱化  
- [ ] AC4. 可取消高亮；用户可拖动分隔线并重新识别  
- [ ] AC5. 本工具导出样例可用；至少一种常见第三方版式可调分隔后可用（或文档说明限制）  
- [ ] AC6. build + lint 通过；图纸仅本地处理；无 OCR 依赖
