# 空格去底 + 图片处理模式

## Goal

解决人物照「满屏背景豆」、以及「已是卡通还被错误风格化」的问题：

1. 图纸支持 **空格（不铺豆）**。
2. 支持 **透明 PNG** 与 **原图点击去背景**（容差可调）。
3. **照片 / 插画** 两模式：切换时应用各自默认参数包；插画不强制去底、不引入卡通化算法。

原则：**不承诺一次完美；保证可开关、可调参、可复现、纯前端。**

## Background

当前（代码确认）：

- `PatternCell = BeadColor`，无空格；每个格子必有豆。
- `createPattern`：缩放 → adjustments → 可选 median-cut → 最近色。
- 左侧风格预设仅为 B/C/S，无法去背景。
- 原图视图已有 `view: 'pattern' | 'source'` + `<img>`，可作取样落点。

## Scope（MVP = 方案 A）

**做：**

- 空格 `BeadColor | null` 贯穿生成 / 绘制 / 导出图例 / 用量 / 高亮 / 总豆数
- PNG alpha → 空格
- 原图点击取样 + 容差去底（可开关）
- 照片 | 插画模式，切换即应用默认包

**不做：** 人像分割模型、画笔修 mask、AI/独立卡通化滤波、PDF

## Requirements

### R1：空格 cell

- **R1.1** `PatternCell = BeadColor | null`；`null` = 不铺豆。
- **R1.2** `counts`、导出图例、用色种数 **不计** 空格；总豆数 = 非空格子数；可显示「空 N 格」。
- **R1.3** draw/export：空格用浅底（如浅灰/棋盘点缀择一，实现定），不写色号；高亮跳过空格。
- **R1.4** median-cut 仅基于非空候选像素（或先 mask 再 quantize），避免背景色占用限色名额（实现见 design）。

### R2：透明与点击去背景

- **R2.1** 缩放后像素 `alpha < 阈值`（如 16）→ 空格；与去底开关无关，始终生效。
- **R2.2** 用户在 **原图预览** 上点击取样 RGB；结合容差，在生成管线中将「接近样本色」的像素标为空格。
- **R2.3** 去底可开关；容差滑块 **0–100**，默认 **32**。
- **R2.4** 「点击原图去除背景」：打开去底 + 进入取样态 + 自动 `view=source` + 提示文案；取样成功后防抖重生成，可回到图纸视图。
- **R2.5** 关闭去底：忽略点击样本；alpha 透明仍为空格。清除样本或重新上传应重置样本色。

### R3：处理模式

- **R3.1** 模式：`photo`（照片）| `illustration`（插画）。
- **R3.2** **切换模式立即应用该模式默认包**（会覆盖当前 adjustments / maxColors / 去底相关默认；取样色可保留或清空——推荐切换时 **保留已取样色** 但按新模式设置 `bgRemoveEnabled` 默认）。
- **R3.3** 默认包（可微调）：

| 项 | 照片 | 插画 |
|----|------|------|
| 去底开关 | 开（需点原图才有样本；无样本时仅 alpha 生效） | 关 |
| maxColors | 24 | 16 |
| adjustments | 照片预设 (0, +10, +10) | 原图 (0,0,0) |

- **R3.4** 无独立卡通化算法；模式 ≠ AI 风格化。
- **R3.5** 用户切换后仍可手改任意参数。

### R4：文案与约束

- **R4.1** 标明去底为辅助，复杂背景请调容差或换透明底图。
- **R4.2** 纯前端，无服务器上传，无新重型依赖。

## Acceptance Criteria

- [ ] AC1：透明 PNG 透明区域在图纸中为空格，不计入用量与总豆数。
- [ ] AC2：原图点击背景 + 容差后，接近背景的格子为空格；主体仍为豆子（简单纯色底可验证）。
- [ ] AC3：关闭去底后，点击样本不再产生空格（alpha 空格仍在）。
- [ ] AC4：draw/export/图例/高亮正确处理空格；导出图例仅统计实豆。
- [ ] AC5：照片/插画切换后参数变为该模式默认包；之后可手改。
- [ ] AC6：插画默认不去底、原图 adjustments；照片默认去底开 + 照片预设 + maxColors 24。
- [ ] AC7：UI 有容差滑块与去底说明文案。
- [ ] AC8：`npm run build` 通过；无新运行时依赖。

## Out of Scope

- ONNX/人像分割、画笔修 mask、posterize/平滑独立算法、多点取样、AI 卡通、PDF、制作模式

## Decisions

| # | 决策 |
|---|------|
| 1 | MVP = 空格 + alpha + 点击去底 + 两模式（方案 A） |
| 2 | 取样在 **原图预览** |
| 3 | 切换模式 **立即应用默认包** |
| 4 | 容差 0–100，默认 32 |

## Constraints

- 保持现有色卡 / median-cut / 图例 / 高亮能力兼容空格
- 中文 UI
- 防抖重生成模式与现有 `scheduleGenerate` 一致
