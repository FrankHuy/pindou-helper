# 色卡重构 + 最大颜色数控制 + 图片调整参数

## Goal

将拼豆图纸助手从「能生成图」升级为「能控制质量的工具」：
1. 重构色卡数据结构，替换为真实 MARD 291 色完整色卡，支持分层范围 + 商家套装预设 + 禁用色。
2. 增加最大颜色数控制（median-cut 降色）。
3. 增加图片基础调整参数（亮度 / 对比度 / 饱和度 + 预设）。

三件事在一个特性分支上完成后合并回 main。

## Background & Motivation

源自 `requirements/competitive-research-and-priorities.md` 中的 P0-1、P0-3、P0-4：

- P0-1：当前 `palette.ts` 只有 33 个自编 MVP 近似色，编号体系和真实 MARD 色号完全不一致（旧 A1 标"白色"#F7F7F2，真实 MARD A1 是浅黄 #FAF4C8）。色卡是整个产品地基，必须换成真实色号。
- P0-3：拼豆不是普通像素画，颜色越多越难买越难摆；照片转拼豆必须能限制颜色数量。
- P0-4：用户上传照片质量差异大，亮度/对比度/饱和度实现成本低但效果提升明显。

### 数据来源

色卡数据来自 `requirements/mard-color-chart.json`（pixel-beads.com 2026 修订版抓取，291 色完整色号 + HEX/RGB），以及 `requirements/mard-merchant-packs.json`（11 个常见商家套装 24/48/72/96/120/144/168/192/216/221/264 的完整 code 列表）。分层逻辑参考 `requirements/mard-color-layers.md`。

已知限制（来自数据文件 notes）：
- 无中文色名，当前 `name = code` 占位。
- HEX 为屏幕参考值，不是实物豆绝对真值。
- 商家套装是 pixel-beads 页面筛选结果，不是 MARD 官方强制标准。
- 264 套装缺 C29，且包含部分扩展系列（P/Q/R/T/Y）；不能按"前 N 色"推算，必须按具体 code 列表存。

## Requirements

### R1：色卡数据结构重构（P0-1）

- **R1.1** 色块数据结构：`{ brand, series, code, name, hex, rgb }`。`series` 是系列前缀（A/B/.../ZG），`name` 暂用 `code` 占位（无中文来源）。
- **R1.2** 数据文件组织在 `src/lib/palettes/` 下，色卡数据与色号数据来自 `mard-color-chart.json` 转换的 TS 常量；商家套装来自 `mard-merchant-packs.json` 转换的 TS 常量。结构设计要让后续新增品牌（COCO/漫漫等）只新增数据文件不改逻辑。
- **R1.3** 色卡分层结构（直接对应源页面的三层）：
  - **范围（range）**：`full`（291 色）/ `standard`（221 色 A–H+M）/ `extended`（70 色 P/Q/R/T/Y/ZG）。
  - **商家套装（merchant pack）**：从 24/48/72/96/120/144/168/192/216/221/264 中选一个，按具体 code 列表生效。
  - **系列筛选（series filter）**：在选中的范围/套装内，可按系列进一步筛选（默认全选）；与套装/range 正交。
  - **用户禁用（disabled）**：禁用单色号在面板里勾选后从生效色集中移除。
- **R1.4** UI 选择器：
  - 第一档「范围」三个按钮：完整色号库(291) / 标准系列(221) / 扩展系列(70)。
  - 第二档「商家套装」可选：当用户选择某个套装（如 96）时，生效色 = 该套装 code 列表（覆盖 range 选择，因为套装已经定义了具体子集）。
  - 第三档「系列筛选」：在当前色集基础上按系列过滤。
  - 默认范围 = `standard`(221)；不在初始就启用商家套装。
- **R1.5** 色卡面板展示当前生效颜色数 / 当前层范围总色数（如 "生效 221 / 标准 221"），并标注「屏幕色仅供参考」。
- **R1.6** 用户禁用色切换、色卡范围切换、商家套装切换、系列筛选变化后，图纸自动走防抖重新生成。
- **R1.7** 不再保留旧的自编 33 色 MVP 色卡数据（与新 MARD 编号体系冲突，只会误导）。最后过渡方式是：从 `mard-color-chart.json` 中的 standard 范围内挑出与旧色最接近的若干色作为"默认高亮"在 UI 加载即选中，让轻度用户感受不到陌生色——这步可选，最低限度是直接弃用旧 palette.ts 数据。

### R2：最大颜色数控制（P0-3）

- **R2.1** 新增参数「颜色数量」，预设选项：8 / 16 / 24 / 32 / 48 / 不限，默认不限。
- **R2.2** 用 median-cut quantization 算法：当 maxColors > 0 且当前生效色卡数 > maxColors 时，先对原图像素降色到 maxColors 个代表色，再把每簇代表色映射到色卡中最近色。这样最终图纸颜色数 ≤ maxColors 且颜色都来自色卡。
- **R2.3** 当 maxColors >= 当前生效色卡数时，跳过降色（会让算法退化、生成色数反被色卡限制），直接走原本每像素匹配色卡路径。
- **R2.4** 在色卡面板显示当前生效颜色数 / 总色卡数 / 最大颜色限制。
- **R2.5** 颜色数量变化后防抖重新生成。

### R3：图片基础调整参数（P0-4）

- **R3.1** 新增三个滑块：亮度（-100 ~ +100，默认 0）、对比度（-100 ~ +100，默认 0）、饱和度（-100 ~ +100，默认 0）。
- **R3.2** 调整在 Canvas 像素级别完成，在 `createPattern` 内部、色卡匹配之前对 ImageData 应用调整。
- **R3.3** 新增「一键预设」：照片 / 头像 / 卡通 / Logo（调整对应参数组合，覆盖三个滑块值）；外加「原图」重置。
- **R3.4** 参数变化后防抖重新生成。
- **R3.5** 预设和手动滑块都要反映在 UI 上（选择预设→滑块跳到对应值；手动拖滑块→取消高亮预设）。

### R4：分支与合并

- **R4.1** 从 main 创建特性分支 `feature/palette-quality-controls`。
- **R4.2** 完成后合并回 main（fast-forward 或 no-ff 由实现者把握，保留清晰历史）。
- **R4.3** 合并前确保 `npm run build` 通过、类型检查通过、lint 通过。

## Acceptance Criteria

- [ ] AC1：`src/lib/palettes/` 下色卡数据从 `mard-color-chart.json` 转换而来，brand/series/code/name(hex)/hex/rgb 字段齐全；新增品牌只需新增数据文件不改逻辑。
- [ ] AC2：商家套装从 `mard-merchant-packs.json` 转换而来，选 24/48 ... /264 中任一档后生效色集等于该套装 code 列表；count 与 codes.length 一致（已由源数据校验）。
- [ ] AC3：UI 可切换范围（完整/标准/扩展）、商家套装、系列筛选、禁用色，四层任意组合都能正确算出"生效色集"，切换后图纸自动重新生成且颜色统计更新。
- [ ] AC4：用户可禁用指定色号，禁用后重新生成图纸排除该色，色卡面板显示禁用态。
- [ ] AC5：颜色数量参数可设为 8/16/24/32/48/不限，设为限定值时生成的图纸颜色数 ≤ 限定值。
- [ ] AC6：亮度/对比度/饱和度滑块可用，调整后图纸可见变化；四个预设可选择且正确设置滑块值，「原图」可重置为 (0,0,0)。
- [ ] AC7：所有调参变化（宽度 / 颜色数 / 图片调整 / 色卡四层切换 / 禁用色）触发防抖重新生成，无重复请求竞态。
- [ ] AC8：色卡面板显示「屏幕色仅供参考」提示（对应数据源 disclaimer）。
- [ ] AC9：`npm run build` 成功，TypeScript 类型检查通过，无 lint 报错。
- [ ] AC10：变更在 `feature/palette-quality-controls` 分支上完成，合并回 main。

## Constraints

- 纯前端，不引入后端、不引入新运行时依赖（median-cut 自行实现）。
- 图片处理全部在浏览器 Canvas / ImageData 本地完成。
- 保持已有的 PWA 功能不受影响。
- 不破坏现有 PNG 导出功能。
- 代码风格遵循项目现有 TypeScript + React + Vite 约定。
- UI 文案使用中文，与现有界面一致。
- 不强制补全中文色名（无来源），`name` 用 `code` 占位即可。

## Out of Scope

- PDF 导出（P0-2，后续单独任务）。
- CIEDE2000 色差算法（P1-1，后续任务）。
- 透明 / 背景移除 / BeckColor|null 改造（P1-2，后续任务）。
- 单格编辑器（P1-3）。
- 制作模式（P1-4）。
- 中文色名补全（数据来源未知，单独事项）。
- 264 套装的缺 C29 异常处理（只按 code 列表如实展示，不做特殊标注）。
