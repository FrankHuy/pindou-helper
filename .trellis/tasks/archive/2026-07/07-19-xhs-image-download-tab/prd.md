# 小红书高清图下载 Tab

## Goal

在拼豆图纸助手中增加一个**与现有拼豆流程完全独立的新 Tab**：用户粘贴公开的小红书分享链接，解析帖内全部高清图，以网格预览 → 点击放大 → **逐张保存**到手机/电脑。

## Background

- 已有可用 demo：`scripts/xhs_image_demo.py`（拉公开分享页 → 解析 `window.__INITIAL_STATE__` → 选 `WB_DFT`/`WB_ORI`/`WB_PRV` → 带 Referer 下图）
- 应用为 Vite + React + TS PWA，Cloudflare 部署（`@cloudflare/vite-plugin` + wrangler）；当前无后端 API
- 浏览器无法直连小红书页面（CORS/反爬）→ 必须服务端代理
- UI 现状：单页 `App.tsx`，无 Tab；用户要求新 Tab 与拼豆功能独立

## Requirements

### Must

- R1. 顶层 Tab 切换：「拼豆图纸」与「小红书下图」；状态互不覆盖
- R2. 粘贴公开分享链接并发起解析（支持 `xiaohongshu.com` / `xhslink.com` 等 demo 可跟随的短链）
- R3. 成功后显示标题 + 高清图缩略图网格
- R4. 点击缩略图进入放大视图；提供**显式「保存图片」按钮**，并有**文字提示可长按图片用系统菜单保存**（两种方式都提示）
- R5. 高清策略与 demo 一致：优先 `WB_DFT` → `WB_ORI` → `WB_PRV`，再回退 `urlDefault`/`url`/`urlPre`
- R6. 不可用链接给出中文错误（需登录、非图文、解析失败、网络失败、非法域名等），不白屏
- R7. 合规文案：仅公开帖、用户有权保存的素材；不提供登录/Cookie 绕过
- R8. 加载中禁用重复提交并显示加载状态
- R9. 后端为 Cloudflare Worker；图片预览/下载经同源代理（带 Referer），避免前端直链跨域失败

### Should

- R10. 放大视图可在多图间切换（上一张/下一张或左右滑），便于连续保存
- R11. 从分享文案中粘贴时，尽量自动提取首个有效 URL

### Out of Scope (MVP)

- ZIP 打包下载
- 登录态 / 私密帖
- 视频正文下载
- 与拼豆流水线联动（一键导入图纸）
- 服务端持久化历史
- 以 Python 脚本作为线上运行时

## Decisions

1. **后端**：Cloudflare Worker（与现有 wrangler 一体），将 demo 逻辑移植为 TS
2. **下载**：不做 ZIP；网格预览 → 放大 → 逐张保存（手机优先）
3. **保存交互**：显式「保存图片」按钮 + 长按系统菜单，**两种方式均有文字提示**
4. **错误策略（默认）**：短链/重定向/解析失败统一中文错误 + 「重试」按钮；不自动无限重试；不尝试绕过登录墙

## Acceptance Criteria

- [ ] AC1. 可在两个 Tab 间切换；拼豆状态与下图状态互不覆盖
- [ ] AC2. 粘贴可用公开图文分享链后，显示标题与 ≥1 张缩略图
- [ ] AC3. 点击缩略图可放大；放大视图有「保存图片」按钮，且有长按保存的文字说明
- [ ] AC4. 点「保存图片」可触发设备侧保存/下载（经同源代理的高清图）
- [ ] AC5. 无效/需登录/非图文/网络失败有中文错误，可重试，不白屏
- [ ] AC6. 拼豆 Tab 原有上传→生成→导出路径无回归
- [ ] AC7. `npm run build` 与 `npm run lint` 通过；`npm run preview`（wrangler）下 API 可联调

## Notes

- 合规边界：只服务公开可访问内容；失败时提示「帖子可能需登录或不存在」，不引导破解
