# 用户体系与AI成本治理分期

## Goal

为拼豆助手建立「先账号、后等级」的身份与运营基础，并在 AI 入口用多层限制避免外部模型 API 成本被打爆：邮箱密码账号 + 反批量注册 + 强制登录配额 + 全局熔断 + 同域极简管理页；完整第三方 Admin 与付费 VIP 后置，数据模型预留。

## Background

- 产品：拼豆图纸 / 工作间（本地）+ 小红书下图（Worker API）；后续将加有成本的 AI 能力。
- 栈：React + Vite SPA，Cloudflare Workers（`worker/index.ts` 现仅 `/api/config` 与 xhs）；无用户体系。
- 已有 Turnstile 模式可复用。
- 攻击面：爬虫/脚本调 AI，以及批量注册多号薅免费额度。

## Confirmed Decisions

| 主题 | 决策 |
|------|------|
| 分期 | Phase 1 账号+护栏+极简管理页 → Phase 2 VIP/付费权益 → Phase 3 完整 Admin 选型落地 |
| 注册 | 邮箱 + 密码；域名 **白名单** 仅 `qq.com` / `gmail.com` / `frankiehu.top`（可配置） |
| 验证 | 必须邮箱验证后才能调 AI；允许未验证登录但锁 AI |
| 人机 | 注册/登录 Turnstile；登录另加防爆破 |
| 反多号 | 注册：IP+浏览器指纹频控；使用：每账号配额 + IP/指纹日关联收紧（非硬共享池） |
| AI 鉴权 | **强制登录**，无匿名额度 |
| 配额 | 按成功次数/日；默认 **3 次/账号/日**；IP/指纹日关联上限（建议 10）；**全局日熔断**；预留月熔断 |
| 底座 | Workers + **D1** 权威数据；KV 仅短缓存且慎写（Free KV 写仅约 1000/日） |
| CF 费用 | Free 可起步非无限；Workers 10 万请求/日等；真风险在上游 AI 账单 |
| 登录 | 邮箱+密码；须忘记/重置密码；Web Crypto 可行哈希（见 design） |
| 运营 | Phase 1 **同域极简管理页**（非仅脚本） |
| 角色 | 四级完整矩阵 Phase 1 落地；VIP **配额差可先与普通相同**，字段与权限分支预留 |
| 第三方 | Auth **不**用 Clerk/Supabase 全家桶；完整 Admin 第三方选型 **Phase 3** |

## Role Permission Matrix（选项 A）

| 能力 | 普通 | VIP | 管理员 | 超管 |
|------|:----:|:---:|:------:|:----:|
| 注册登录 / 本地工具 | ✓ | ✓ | ✓ | ✓ |
| 验证后调 AI（按配额） | ✓ | ✓（可更高） | ✓ | ✓ |
| 进极简管理页 | | | ✓ | ✓ |
| 搜用户、看日用量 | | | ✓ | ✓ |
| 封禁/解封 普通与 VIP | | | ✓ | ✓ |
| 调单用户当日额度 | | | ✓ | ✓ |
| 查看/开关全局熔断 | | | ✓ | ✓ |
| 改邮箱域名白名单 | | | | ✓ |
| 改他人角色（含管理员） | | | | ✓ |
| 动超管账号 | | | | 受限自操作/设计约定 |

- 公开注册默认 `role=user`（普通）；角色不可自选。
- 初始超管：环境变量指定邮箱，验证/首次匹配后提升为 `super_admin`。
- 管理员不可提拔管理员、不可操作超管。

## Requirements

### R1 — Phase 1 交付

1. 邮箱+密码注册/登录/登出/会话；忘记密码。
2. 注册：Turnstile + 域名白名单 + 邮箱验证邮件 + IP/指纹注册频控。
3. AI 网关契约（可先接 mock/占位上游）：强制登录、邮箱已验证、未封禁、每账号日配额、IP/指纹关联上限、全局日熔断；统一错误码。
4. D1 用户/会话/用量/配置模型；`role`/`plan`/配额相关字段预留。
5. 同域极简管理页：上述管理员/超管能力。
6. 可配置项：白名单、默认 3/日、关联上限、全局日上限、熔断开关。

### R2 — Phase 2

- VIP/付费开通与配额差；支付提供方另定。
- 不阻塞 Phase 1。

### R3 — Phase 3

- 完整用户运营后台；优先评估第三方 Admin 挂现有 API；可替换极简页。

### R4 — 约束

- 不破坏本地图纸/工作间「主路径不上传」承诺；AI 上传须显式。
- 兼容 `wrangler.jsonc` Worker + SPA；`/api/*` worker first。
- 计数权威在 D1，避免 Free KV 写打满。
- 指纹为软信号，不作唯一信任根。

## Out of Scope

- Phase 1 不上支付与 VIP 开通闭环（仅预留）。
- 不实现具体 AI 模型业务效果（智能分色等另任务）；本任务提供身份与成本护栏。
- 小红书下载不强制登录。
- 不用浏览器指纹当唯一身份。

## Open Questions

（产品决策已收敛；下列仅实现期细节，由 design 默认并在评审可改）

1. 日界时区：默认 **UTC**（与 CF Free 限额重置一致），展示可本地化。
2. 扣次时机：默认 **上游成功计费后扣次**；若先扣后补偿需在实现注明。
3. 邮件提供商：默认 Resend（或等价），密钥走 Worker Secret。
4. 指纹库：默认轻量客户端指纹（非商业 FingerprintJS Pro），服务端只存哈希。
